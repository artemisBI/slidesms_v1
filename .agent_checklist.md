## ðŸ¤” Ask More Questions

**When to ask clarifying questions:**
- User's request is vague or could be interpreted multiple ways
- Multiple valid approaches exist (ask for preference)
- Design decisions that affect user experience
- Technical trade-offs (performance vs. simplicity)
- User forgets to provide complete context

**Example questions to ask:**
- "Would you prefer [Option A] or [Option B]?"
- "Should this section be on a light or dark background?"
- "Do you want this animation to be subtle or more prominent?"
- "Is this feature critical for MVP, or can we add it later?"

## ðŸ› Debugging Visual Issues

When the user reports a visual problem (e.g., "text is hard to read", "colors are wrong"):

**Best approach:**
1. **Ask for specifics**: "Which section?" or "Can you share a screenshot?"
2. **Request CSS class**: "Can you inspect the element and share the class name?"
3. **Use diagnostic mode**: Temporarily change the background to white/transparent to isolate the issue

**Example user feedback that works well:**
> "The Features section (2nd section, 'Why SlideSMS?') has white text on a dark background that's hard to read. Chrome DevTools shows `.features` has `background: #0A0A0F`. Can you lighten this?"

**What to avoid:**
- Guessing which element is causing the issue
- Making multiple changes at once (hard to diagnose)
- Not asking for clarification when the issue is vague
